# PDF 문제 라벨링 시스템 UI/UX 개선 연구 리포트

**작성일**: 2025-11-26
**버전**: 1.0
**대상 시스템**: PDF 문제 이미지 자동 크롭 + 라벨링 데스크톱 앱

---

## Executive Summary

코드베이스 분석 및 경쟁 도구 벤치마킹을 통해 다음 핵심 발견사항을 도출했습니다:

1. **반복 입력의 비효율성**: 매 그룹마다 책이름, 과정, 페이지, 문항번호를 입력해야 하며, 자동완성 기능이 있지만 한계가 있음
2. **키보드/마우스 전환 빈도**: 블록 선택(마우스) → 그룹 생성(마우스) → 정보 입력(키보드) → 다음 블록 선택(마우스) 패턴으로 전환이 잦음
3. **Undo/Redo 부재**: 그룹 생성 후 실수 시 삭제만 가능하고, 작업 이력 관리가 없음
4. **일괄 처리 미지원**: 여러 페이지의 그룹을 한 번에 내보내기는 가능하나, 일괄 편집은 불가
5. **시각적 피드백 개선 여지**: 작업 완료 페이지 vs 미완료 페이지 구분이 명확하지 않음

### 1.2 소요 시간 분석

한 페이지에 4개 문제가 있다고 가정할 때:

| 작업 | 예상 소요 시간 | 누적 시간 |
|------|--------------|----------|
| 페이지 로드 | 1-2초 | 2초 |
| 첫 번째 그룹 블록 선택 | 5-10초 | 12초 |
| 그룹 생성 버튼 클릭 | 1초 | 13초 |
| 편집 버튼 클릭 | 1초 | 14초 |
| 문항 정보 입력 (책이름, 과정, 페이지, 번호) | 10-15초 | 29초 |
| 저장 확인 | 1초 | 30초 |
| **2~4번째 그룹 반복 (x3)** | 약 45초 | **75초** |
| 다음 페이지 이동 | 1초 | 76초 |

**결론**: 한 페이지당 약 1분 15초, 100페이지 문서 기준 약 2시간 소요

### 1.3 인지 부하 분석

현재 시스템의 인지 부하를 평가합니다:

| 항목 | 부하 수준 | 설명 |
|------|----------|------|
| 블록-문제 매핑 인지 | **높음** | 어떤 블록들이 하나의 문제를 구성하는지 판단 필요 |
| 컬럼 구분 (L/R) | 중간 | 시스템이 자동 감지하나 확인 필요 |
| 문항번호 순서 기억 | 중간 | 이전 문항번호를 기억하고 다음 번호 입력 |
| 책/과정 정보 기억 | 낮음 | 자동완성으로 해결됨 |
| 페이지 진행 상황 파악 | **높음** | 어느 페이지까지 완료했는지 한눈에 파악 어려움 |
| 그룹 경계 시각화 | 중간 | 색상으로 구분되나 복잡한 페이지에서 혼란 가능 |

## 2. 불편 지점 (Pain Points)

### 2.1 반복 작업

**현상**:
- 매 그룹마다 문항 정보 입력 필요
- 문항번호가 순차적임에도 수동 입력 (1, 2, 3, 4...)
- 페이지 번호가 자동 계산되지만 확인/수정에 시간 소요

**코드 근거** (`GroupPanel.tsx` 43-50행):
```typescript
const startEditing = (group: ProblemGroup) => {
  setEditingGroupId(group.id);
  setEditForm({
    bookName: group.problemInfo?.bookName || defaultBookName,
    course: group.problemInfo?.course || defaultCourse,
    page: group.problemInfo?.page || bookPage || 1,
    problemNumber: group.problemInfo?.problemNumber || '',  // 매번 빈 값
  });
};
```

**개선 포인트**: `problemNumber`가 빈 값으로 시작하므로 매번 입력 필요

### 2.2 인지 부하

**현상**:
- 그룹 생성 후 문항 정보 편집을 위해 별도 클릭 필요
- 현재 어떤 블록이 어떤 그룹에 속하는지 즉시 파악 어려움
- 페이지별 작업 완료 상태가 사이드바에 표시되지 않음

**코드 근거** (`PageViewer.tsx` 191-225행):
- 그룹 생성과 정보 입력이 분리된 워크플로우
- 그룹 생성 시 `problemInfo`가 비어있음

### 2.3 에러 취약점

**현상**:
- **Undo/Redo 없음**: 잘못된 그룹 생성 시 삭제 후 재생성 필요
- 블록 선택 실수 시 처음부터 다시 선택해야 함
- 페이지 이동 시 현재 작업 내용 저장 확인 없음 (2초 자동 저장에 의존)

**코드 근거** (`PageViewer.tsx` 114-125행):
```typescript
// 자동 저장 (디바운스: 2초)
useEffect(() => {
  if (!groupsData) return;
  const timer = setTimeout(() => {
    saveGroups(localGroups);
  }, 2000);
  return () => clearTimeout(timer);
}, [localGroups]);
```
- 2초 이내 페이지 이동 시 저장 누락 가능성

### 2.4 입력 번거로움

**현상**:
- 키보드 단축키 제한적 (화살표 키만 지원)
- 그룹 생성을 위한 버튼 클릭 필수
- 다음 문제로 이동하는 빠른 방법 없음

**코드 근거** (`PageViewer.tsx` 128-172행):
```typescript
const handleKeyDown = (e: KeyboardEvent) => {
  switch (e.key) {
    case 'ArrowLeft':  // 이전 페이지
    case 'ArrowRight': // 다음 페이지
    case 'Delete':     // 그룹 삭제
    case 'Escape':     // 선택 해제
    // 그룹 생성 단축키 없음!
  }
};
```

## 3. UI/UX 개선 제안

### 3.1 입력 효율성

#### 3.1.1 스마트 문항번호 자동 증가

**제안**: 마지막 생성된 그룹의 문항번호를 파싱하여 다음 번호 자동 제안

```typescript
// 예시 로직
const getNextProblemNumber = (lastNumber: string): string => {
  // "3" → "4"
  // "예제2" → "예제3"
  // "유형01" → "유형02"
  const match = lastNumber.match(/(\D*)(\d+)$/);
  if (match) {
    const prefix = match[1];
    const num = parseInt(match[2], 10) + 1;
    return `${prefix}${num}`;
  }
  return '';
};
```

#### 3.1.2 키보드 단축키 확장

| 단축키 | 동작 |
|-------|------|
| `G` | 선택된 블록으로 새 그룹 생성 |
| `E` | 마지막 생성 그룹 편집 모드 진입 |
| `Enter` (편집 중) | 저장 후 다음 그룹 생성 대기 |
| `Tab` (편집 중) | 다음 입력 필드로 이동 |
| `Ctrl+Z` | Undo |
| `Ctrl+Shift+Z` | Redo |
| `1-9` | 해당 번호 블록 토글 선택 |

#### 3.1.3 퀵 라벨링 모드

**제안**: 책이름, 과정이 고정된 상태에서 문항번호만 빠르게 입력하는 모드

```
[퀵 라벨링 모드 ON]
- 블록 선택 → 숫자 키 입력 → 자동 그룹 생성 + 저장
- 예: 블록 선택 후 "3" 입력 → "책이름 - 과정, 123p, 3" 자동 생성
```

### 3.2 시각적 피드백

#### 3.2.1 페이지 상태 인디케이터

사이드바 또는 네비게이션에 페이지별 상태 표시:

| 상태 | 표시 | 설명 |
|------|------|------|
| 미작업 | 회색 원 | 그룹 0개 |
| 진행 중 | 주황색 원 | 그룹 있으나 문항정보 미입력 |
| 완료 | 초록색 체크 | 모든 그룹에 문항정보 입력됨 |

#### 3.2.2 블록-그룹 시각화 개선

**현재**: 모든 그룹이 같은 초록색
**제안**: 그룹별로 다른 색상 할당 (L1=파랑, L2=초록, R1=보라 등)

#### 3.2.3 미리보기 패널

그룹 생성 시 크롭될 이미지 미리보기 표시

### 3.3 작업 흐름

#### 3.3.1 원클릭 그룹 생성 + 편집

**현재 흐름**:
1. 블록 선택
2. "그룹 생성" 버튼 클릭
3. "편집" 버튼 클릭
4. 정보 입력

**제안 흐름**:
1. 블록 선택
2. "그룹 생성" 버튼 클릭 (자동으로 편집 모드 진입, 번호 자동 제안)
3. 정보 입력 (Enter로 저장)

#### 3.3.2 Undo/Redo 시스템

최근 20개 작업에 대한 Undo/Redo 지원:
- 그룹 생성
- 그룹 삭제
- 그룹 정보 수정
- 블록 선택 변경

#### 3.3.3 일괄 편집 모드

여러 그룹의 공통 정보(책이름, 과정)를 한 번에 수정

### 3.4 학습 곡선

#### 3.4.1 온보딩 투어

첫 사용 시 주요 기능 안내:
1. "이것은 자동 검출된 블록입니다"
2. "클릭하거나 드래그하여 선택하세요"
3. "선택 후 이 버튼으로 그룹을 만드세요"
4. "문항 정보를 입력하세요"

#### 3.4.2 키보드 단축키 힌트

현재 가능한 동작의 단축키를 화면 하단에 표시:
```
[G] 그룹 생성  |  [Delete] 삭제  |  [←→] 페이지 이동  |  [Esc] 선택 해제
```

#### 3.4.3 도움말 패널

`?` 키로 전체 단축키 및 사용법 오버레이 표시

### 3.5 에러 방지

#### 3.5.1 저장 상태 강화

페이지 이동 전 미저장 변경사항 확인:
```
"저장되지 않은 변경사항이 있습니다. 페이지를 이동하시겠습니까?"
[저장 후 이동] [이동] [취소]
```

#### 3.5.2 유효성 검사

- 문항번호 중복 검사
- 빈 그룹 (블록 0개) 방지
- 문항정보 미입력 시 경고 표시

#### 3.5.3 자동 저장 표시기 개선

현재: 마지막 저장 시간만 표시
제안: 저장 대기 중(2초 카운트다운), 저장 중, 저장 완료 상태 표시

## 4. 구체적 기능 제안 (우선순위별)

### Priority 5 (최우선)

#### 4.1 그룹 생성 시 자동 편집 모드 진입

**문제점**: 그룹 생성 후 별도로 편집 버튼을 클릭해야 함
**해결책**: 그룹 생성 즉시 해당 그룹의 편집 폼을 열고 문항번호 필드에 포커스
**구현 난이도**: Low
**영향도**: 매 그룹당 2번의 클릭 절감, 키보드 유지
**예상 공수**: 0.5일

```typescript
// PageViewer.tsx 수정
const handleCreateGroup = () => {
  // ... 기존 그룹 생성 로직
  const newGroup = { id: newGroupId, ... };
  setLocalGroups([...localGroups, newGroup]);

  // 추가: 자동 편집 모드 진입
  setTimeout(() => {
    startEditing(newGroup);  // GroupPanel에 전달
  }, 100);
};
```

#### 4.2 문항번호 자동 증가 제안

**문제점**: 순차적 문항번호(1,2,3...)를 매번 수동 입력
**해결책**: 마지막 입력 번호 파싱 → 다음 번호 자동 채움
**구현 난이도**: Low
**영향도**: 문항번호 입력 시간 80% 감소
**예상 공수**: 0.5일

#### 4.3 그룹 생성 키보드 단축키 (G)

**문제점**: 마우스로 버튼 클릭 필수
**해결책**: 선택된 블록이 있을 때 G 키로 그룹 생성
**구현 난이도**: Low
**영향도**: 키보드-마우스 전환 50% 감소
**예상 공수**: 0.25일

### Priority 4

#### 4.4 Undo/Redo 기본 구현

**문제점**: 실수 시 복구 불가, 삭제 후 재생성 필요
**해결책**: 그룹 생성/삭제/수정에 대한 Undo/Redo 스택
**구현 난이도**: Medium
**영향도**: 실수 복구 시간 90% 감소, 심리적 안정감 증가
**예상 공수**: 2일

```typescript
interface HistoryState {
  groups: ProblemGroup[];
  timestamp: Date;
}

const [history, setHistory] = useState<HistoryState[]>([]);
const [historyIndex, setHistoryIndex] = useState(-1);
```

#### 4.5 페이지별 작업 상태 인디케이터

**문제점**: 어느 페이지까지 작업했는지 파악 어려움
**해결책**: 페이지 네비게이션에 상태 표시 (미완료/진행중/완료)
**구현 난이도**: Medium
**영향도**: 작업 계획 및 진행 파악 용이
**예상 공수**: 1일

#### 4.6 퀵 저장 단축키 (Ctrl+S)

**문제점**: 2초 자동 저장에만 의존
**해결책**: Ctrl+S로 즉시 저장 + 시각적 확인
**구현 난이도**: Low
**영향도**: 저장 불안감 해소
**예상 공수**: 0.25일

### Priority 3

#### 4.7 그룹별 색상 구분

**문제점**: 여러 그룹이 같은 색상으로 구분 어려움
**해결책**: L1=파랑, L2=청록, R1=보라, R2=분홍 등 자동 할당
**구현 난이도**: Low
**영향도**: 시각적 구분 명확화
**예상 공수**: 0.5일

#### 4.8 크롭 미리보기 패널

**문제점**: 최종 결과물 확인 위해 내보내기 필요
**해결책**: 그룹 선택 시 오른쪽 패널에 크롭 예상 이미지 표시
**구현 난이도**: Medium
**영향도**: 결과 확인 편의성
**예상 공수**: 1.5일

#### 4.9 키보드 단축키 힌트 바

**문제점**: 사용 가능한 단축키 모름
**해결책**: 화면 하단에 현재 상황에서 가능한 단축키 표시
**구현 난이도**: Low
**영향도**: 학습 곡선 완화
**예상 공수**: 0.5일

### Priority 2

## 5. 경쟁 시스템 벤치마킹

### 5.1 Label Studio

[Label Studio](https://labelstud.io/)는 오픈소스 데이터 라벨링 도구로, 다음 UX 패턴이 우수합니다:

**키보드 단축키 시스템**
- 모든 라벨에 자동 단축키 할당 (1, 2, 3...)
- 커스텀 단축키 정의 가능 (`hotkey="ctrl+1"`)
- [Ctrl+Enter]로 즉시 제출
- [Backspace]로 영역 삭제

**참고**: [Label Studio Hotkeys 문서](https://labelstud.io/guide/hotkeys)

**배울 점**:
- 라벨별 단축키를 UI에 표시
- 커스터마이즈 가능한 단축키 설정
- 작업별 최적화된 키보드 레이아웃

### 5.2 CVAT

[CVAT](https://www.cvat.ai/)는 비디오/이미지 어노테이션 도구로, 전문 데이터 어노테이터 피드백 기반 UI입니다:

**작업 저장 UX**
- "Save Work" 명시적 클릭 필요 (자동 저장 아님)
- 저장 상태 명확히 표시
- [N] 키로 새 어노테이션 생성

**참고**: [CVAT Documentation](https://docs.cvat.ai/v2.18.0/docs/)

**배울 점**:
- 명시적 저장과 자동 저장의 조합
- 상태 표시 명확화
- 전문가 피드백 기반 UI 개선

### 5.3 Labelbox

[Labelbox](https://labelbox.com/)는 엔터프라이즈급 라벨링 플랫폼으로, 효율성에 집중합니다:

**Model-Assisted Labeling**
- AI가 pre-label 제공 → 사용자는 수정만
- 자동화 효율성 점수 제공
- 이미지당 20-30초로 단축 (분 단위에서)

**배치 처리**
- 일괄 작업 적용 기능
- 단계별 리뷰 파이프라인
- 그룹별 데이터 관리

**참고**: [Labelbox Annotate Overview](https://docs.labelbox.com/docs/annotate-overview)

**배울 점**:
- 딥러닝 자동화 도입 시 "수정 중심" 워크플로우
- 자동화 효율성 지표 제공
- 일괄 처리 지원

### 5.4 기타 도구들

**RectLabel / VGG Image Annotator**
- 심플한 UI
- 빠른 박스 그리기
- JSON 내보내기

**Roboflow**
- AI 보조 워크플로우
- 자동 트래킹
- 유사 객체 태깅

### 5.5 배울 수 있는 UX 패턴 요약

| 패턴 | 설명 | 적용 우선순위 |
|------|------|-------------|
| **단축키 시각화** | 가능한 동작의 단축키를 UI에 표시 | 높음 |
| **자동 번호 할당** | 라벨/그룹에 자동 순번 제안 | 높음 |
| **즉각적 피드백** | 작업 결과 즉시 시각화 | 높음 |
| **Pre-labeling** | AI가 먼저 제안, 사용자가 수정 | 중간 (Phase 2) |
| **일괄 처리** | 여러 항목 동시 수정 | 중간 |
| **효율성 지표** | 작업 속도/정확도 표시 | 낮음 |

## 7. 딥러닝 자동화 대비

### 7.1 검수 UI 설계

향후 딥러닝 모델이 자동으로 그룹을 생성할 때의 UI:

**자동 그룹 표시**
```
┌─────────────────────────────────────┐
│ [AI 제안] L1 - 신뢰도 95%           │
│ ├ 블록 1, 2, 3                      │
│ └ [승인] [수정] [거부]              │
├─────────────────────────────────────┤
│ [AI 제안] L2 - 신뢰도 78%  ⚠️       │
│ ├ 블록 4, 5                         │
│ └ [승인] [수정] [거부]              │
└─────────────────────────────────────┘
```

**승인 모드**
- 높은 신뢰도(>90%): 기본 승인, 문제시만 수정
- 중간 신뢰도(70-90%): 확인 필요 표시
- 낮은 신뢰도(<70%): 수동 검토 필수 표시

### 7.2 신뢰도 표시 방법

**시각적 표현**
| 신뢰도 | 색상 | 테두리 | 아이콘 |
|--------|------|--------|--------|
| 90%+ | 초록 | 실선 | ✓ |
| 70-90% | 노랑 | 점선 | ⚠️ |
| <70% | 빨강 | 점선 굵게 | ❓ |

**정보 표시**
- 그룹 카드에 신뢰도 퍼센트 표시
- 마우스 오버 시 상세 정보 (어떤 블록이 불확실한지)
- 전체 페이지 신뢰도 요약

### 7.3 빠른 수정 워크플로우

**시나리오별 워크플로우**

**케이스 1: AI가 정확히 예측**
1. 페이지 로드 → AI 그룹 표시
2. 사용자: "전체 승인" 버튼 또는 [Ctrl+Enter]
3. 문항 정보만 빠르게 입력 (자동 제안 활용)
4. 다음 페이지

**케이스 2: 일부 수정 필요**
1. 페이지 로드 → AI 그룹 표시
2. 노란색 표시된 그룹 클릭
3. 블록 추가/제거 (Ctrl+클릭)
4. [Enter]로 확정
5. "나머지 승인" 버튼

**케이스 3: AI 예측 부정확**
1. 페이지 로드 → AI 그룹 표시
2. "모두 거부" 버튼
3. 수동 라벨링 모드로 전환
4. 일반 워크플로우 진행

**단축키 제안**

| 단축키 | 동작 |
|-------|------|
| `Y` | 현재 AI 그룹 승인 |
| `N` | 현재 AI 그룹 거부 |
| `Ctrl+Y` | 페이지 전체 AI 그룹 승인 |
| `Ctrl+N` | 페이지 전체 AI 그룹 거부 |
| `M` | 수동 모드로 전환 |

### 7.4 학습 데이터 피드백 루프

사용자 수정 사항을 학습 데이터로 활용:

```
[AI 예측] → [사용자 수정] → [수정 내용 저장] → [모델 재학습]
```

저장할 정보:
- 원본 AI 예측
- 최종 사용자 결정
- 수정 유형 (블록 추가/제거/그룹 병합/분할)
- 페이지 특성 (컬럼 수, 블록 밀도 등)

## 8. 결론 및 권장사항

### 8.1 핵심 권장사항

1. **즉시 구현 권장 (1-2일)**
   - 그룹 생성 시 자동 편집 모드 진입
   - 문항번호 자동 증가
   - G 키 단축키
   - Ctrl+S 저장

2. **1주일 내 구현 권장**
   - Undo/Redo 기본 기능
   - 페이지 상태 인디케이터
   - 단축키 힌트 바

3. **지속적 개선**
   - 사용자 피드백 수집 시스템
   - 작업 시간 측정 및 개선점 도출
   - 딥러닝 자동화 UI 점진적 도입

### 8.2 성공 지표

| 지표 | 현재 추정 | 목표 |
|------|----------|------|
| 문제당 평균 작업 시간 | 18초 | 8초 |
| 페이지당 평균 작업 시간 | 75초 | 30초 |
| 키보드 단독 작업 가능 비율 | 0% | 80% |
| Undo 사용으로 인한 시간 절감 | - | 실수당 30초 |

### 8.3 향후 발전 방향

**Phase 1 (현재)**: 수동 라벨링 최적화
- 키보드 중심 워크플로우
- 반복 작업 자동화
- 피드백 루프 구축

**Phase 2 (딥러닝 도입)**: AI 보조 라벨링
- 자동 그룹 제안
- 신뢰도 기반 검수 UI
- 인간-AI 협업 워크플로우

**Phase 3 (완전 자동화)**: 인간은 검수만
- 배치 승인 시스템
- 예외 케이스만 수동 처리
- 품질 관리 대시보드
