# 통합 문제은행 시스템 UI/UX 연구 리포트

**날짜**: 2025-11-28
**요청**: PDF 라벨링, HWPX/HML 파싱, 해설 수집, 빠른 답, 문제은행, A4 템플릿을 통합하는 효율적인 UI/UX 설계
**분석 범위**: 현재 시스템 분석, 통합 설계, 리팩토링 제안

---

## 1. 현재 시스템 분석

### 1.1 현재 네비게이션 구조

```
현재 사이드바:
├── 대시보드 (/)
├── 문서 관리 (/documents)      ← PDF만 지원
├── 라벨링 작업 (/labeling)     ← PDF 블록 그룹핑
├── 문제은행 (/problems)        ← 문서별 분리, 필터 제한적
├── 해설 연결 (/solutions)      ← Mock 데이터, 미완성
├── 작업 관리 (/tasks)
├── 통계 (/stats)
└── 설정 (/settings)
```

### 1.2 현재 시스템의 문제점

| 영역 | 문제점 | 심각도 |
|------|--------|--------|
| **입력 소스** | PDF만 지원, HWPX/HML 미지원 | 높음 |
| **문제은행** | 문서별 분리, 통합 검색 불가 | 높음 |
| **해설 시스템** | Mock 데이터, 실제 기능 없음 | 높음 |
| **출력** | 템플릿 시스템 없음 | 높음 |
| **워크플로우** | 단계별 분리, 연결성 부족 | 중간 |
| **메타데이터** | 과목/단원/난이도 태깅 없음 | 중간 |

### 1.3 현재 데이터 흐름

```
현재:
PDF 업로드 → 블록 검출 → 그룹핑 → 문제 이미지 저장 → (끝)
                                          │
                              문서별 분리된 문제은행
```

| **그룹핑** | 수동 (드래그) | 자동 (번호 패턴) |
| **정답 추출** | 별도 입력 필요 | 자동 ([정답] 태그) |
| **해설 추출** | 별도 입력 필요 | 자동 (풀이 섹션) |
| **메타데이터** | 수동 설정 | 파일에서 일부 추출 |
| **저장** | 동일 | 동일 |

  HWPX 업로드 → 자동 파싱 → 문제+정답+해설 자동 분리 → 문제은행 저장
