## 토스 UX 3대 원칙 (CLAUDE.md 기준)

```
┌──────────────────────────────────────────────────────────────┐
│  원칙 1: 기본값의 자동화 (Automation by Default)              │
│  - AI가 모든 필드를 사전 입력                                 │
│  - 사용자는 승인/수정만 수행                                  │
├──────────────────────────────────────────────────────────────┤
│  원칙 2: 시각적 단순화 (Visual Simplicity)                   │
│  - 화면에는 문제 이미지 + AI 태그만 표시                      │
│  - 메타데이터는 Progressive Disclosure (점진적 공개)          │
├──────────────────────────────────────────────────────────────┤
│  원칙 3: 속도감 (Velocity)                                   │
│  - Optimistic UI: 버튼 클릭 즉시 다음 화면                    │
│  - 키보드 중심 워크플로우 (Enter=승인, E=편집, T=태그)         │
└──────────────────────────────────────────────────────────────┘
```

## 1. 현재 UI 요소별 점검

### 1.1 보라색 그라데이션 헤더 ❌ 삭제 권장

```tsx
// PageViewer.tsx L627-641
<div className="bg-gradient-to-r from-blue-600 to-purple-600 rounded-xl p-6 text-white shadow-lg">
  <h2 className="text-2xl font-bold">라벨링 작업</h2>
  <p className="mt-1 text-blue-100">블록을 선택하여 문제 그룹을 생성하세요</p>
  <button>📤 페이지 내보내기</button>
</div>
```

| 토스 원칙 | 평가 | 이유 |
|-----------|------|------|
| 시각적 단순화 | ❌ 위반 | 큰 배너가 핵심 작업 영역(PDF)을 가림 |
| 속도감 | ❌ 위반 | 이미 알고 있는 정보("라벨링 작업")를 반복 |

**권장**: 완전 삭제. "페이지 내보내기" 버튼은 상단 헤더로 이동

---

### 1.2 페이지 네비게이션 ⚠️ 간소화 필요

```
현재 구성:
┌────────────────────────────────────────────────────────────────┐
│ [<<] [< 이전]    PDF [1] /120  |  📖 책 [1p] ⚙️    [다음 >] [>>] │
│ ─────────────────────────────────────────────────────────────── │
│ 진행률                                                    1%   │
│ ■□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□ │
│ ← → 페이지 이동 | G 그룹 생성 | Ctrl+S 즉시 저장 | Esc 선택 해제 │
└────────────────────────────────────────────────────────────────┘
```

| 토스 원칙 | 평가 | 이유 |
|-----------|------|------|
| 시각적 단순화 | ⚠️ 부분 위반 | 정보가 너무 많음 (PDF/책/진행률/단축키 한꺼번에) |
| Progressive Disclosure | ❌ 위반 | 페이지 오프셋 설정이 항상 노출 |

**권장**:
```
간소화된 네비게이션:
┌────────────────────────────────────────────────────────────────┐
│ [< 이전]         1 / 120 (책 15p)          [다음 >]           │
│ ━━━━━━━━━■□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□□ 1%        │
└────────────────────────────────────────────────────────────────┘
- [<<] [>>] 버튼 제거 (거의 안 씀)
- 단축키 힌트는 ? 버튼 클릭 시만 표시
- 페이지 오프셋 설정은 ⚙️ 클릭 시만 드롭다운
```

### 1.3 하단 통계 카드 4개 ❌ 삭제 권장

```tsx
// PageViewer.tsx L700-756
<div className="grid grid-cols-2 md:grid-cols-4 gap-4">
  {/* 전체 블록: 42 */}
  {/* 그룹: 3 */}
  {/* 선택됨: 0 */}
  {/* 저장 상태: 대기 */}
</div>
```

| 토스 원칙 | 평가 | 이유 |
|-----------|------|------|
| 시각적 단순화 | ❌ 심각 위반 | 핵심 작업과 무관한 정보 과다 노출 |
| 속도감 | ❌ 위반 | 스크롤 증가, 작업 영역 축소 |

**권장**: 완전 삭제
- "블록 수"는 사용자에게 의미 없음
- "선택됨"은 캔버스에서 시각적으로 이미 표시됨
- "저장 상태"는 토스트로 충분

### 3.2 삭제 대상 (총 4개 영역)

```diff
- 보라색 그라데이션 헤더 ("라벨링 작업" 배너)
- "페이지 미리보기" 섹션 헤더
- 하단 통계 카드 4개 (전체 블록/그룹/선택됨/저장상태)
- 페이지 오프셋 설정 패널 (숨김 처리, 필요시만 노출)
```

### 3.3 간소화 대상 (총 2개 영역)

```diff
~ 페이지 네비게이션
  - [<<] [>>] 버튼 제거
  - 단축키 힌트 축소 (한 줄로)
  - PDF/책 페이지 통합 표시

~ 상단 헤더
  - 문서명 + 진행률 + 내보내기 버튼만
```

### 3.4 개선 대상 (총 1개 영역)

```diff
! 오른쪽 GroupPanel → AccordionGroupPanel
  - 현재 페이지: 펼쳐진 상태
  - 작업한 페이지: 접힌 상태로 스택
  - 완료 페이지: 체크 아이콘
```

## 4. 원칙별 개선 효과

### 4.1 원칙 1: 기본값의 자동화

| 현재 | 개선 후 |
|------|---------|
| 문항번호 수동 입력 | ✅ 이미 자동 제안 (유지) |
| 책이름/과정 수동 입력 | ✅ 이미 document_id에서 파싱 (유지) |
| 그룹 생성 후 수동 확정 | ✅ 이미 자동 확정 (유지) |

→ **자동화는 이미 잘 되어 있음**

### 4.2 원칙 2: 시각적 단순화

| 현재 | 개선 후 |
|------|---------|
| 화면 요소 10개+ | 화면 요소 5개 이하 |
| 스크롤 필요 | 한 화면에 모두 표시 |
| 정보 과부하 | 핵심만 노출 |

→ **대폭 개선 필요**

### 4.3 원칙 3: 속도감

| 현재 | 개선 후 |
|------|---------|
| 키보드 단축키 ✅ | 유지 |
| Optimistic UI ✅ | 유지 |
| 작업 영역 협소 | 작업 영역 확대 |

→ **작업 영역 확대로 체감 속도 향상**

## 7. 결론

### 현재 문제점
1. **정보 과부하**: 핵심 작업(블록 선택 → 그룹 생성)과 무관한 정보가 너무 많음
2. **공간 낭비**: 불필요한 헤더, 배너, 통계 카드가 작업 영역을 침범
3. **토스 철학 위반**: "시각적 단순화" 원칙에 크게 어긋남

### 권장 조치
```
Phase 40: UI/UX 전면 리뉴얼
├── Phase 40-A: 불필요 요소 삭제 (30분)
├── Phase 40-B: 네비게이션 간소화 (1시간)
└── Phase 40-C: 아코디언 패널 구현 (2시간)

총 예상 시간: 3.5시간
```

### 기대 효과
- **토스 적합도**: 45% → 85%
- **작업 영역**: +30% 확대
- **체감 속도**: 불필요한 시각 요소 제거로 집중도 향상
