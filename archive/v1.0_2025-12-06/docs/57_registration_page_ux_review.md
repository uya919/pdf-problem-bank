## Part 2: 상세 문제점 분석

### 🔴 문제 1: 이중 업로드 영역으로 인한 인지 부하 (심각)

**현상**:
- 페이지에 두 개의 업로드 영역이 존재
- 사용자가 "어디에 파일을 올려야 하지?" 고민 필요
- "또는"이라는 구분선이 있지만 우선순위가 불명확

**위반 원칙**: Visual Simplicity, Velocity

**근거 문서**: `45_zero_base_redesign_report.md`
> "한 화면에 '지금 해야 할 일'이 명확히 보임"

**코드 위치**: [RegistrationPage.tsx:265-321](frontend/src/pages/RegistrationPage.tsx#L265-L321)

---

### 🔴 문제 2: 정보 계층 불명확 (심각)

**현상**:
- DualUploadCard가 상단에 있지만, 기존 단일 업로드가 더 범용적
- 신규 사용자는 듀얼 매칭이 필수인지 선택인지 알 수 없음
- CTA 버튼이 여러 개 (듀얼 매칭 시작, 기존 드롭존)

**위반 원칙**: Progressive Disclosure, 정보 계층 명확화

**권장**: 주 CTA는 하나, 고급 기능은 접힌 상태로 제공

### 🟡 문제 4: 버튼 스타일 불일치

**프로젝트 Button 컴포넌트**:
- `variant="solid"`: 토스 블루 배경
- `variant="ghost"`: 투명 배경
- 토스 디자인 시스템 색상 사용

**DualUploadCard 버튼**:
```typescript
// 직접 스타일링 (컴포넌트 미사용)
className="bg-gradient-to-r from-blue-600 to-purple-600"
```

| 문제점 | 설명 |
|--------|------|
| 그라데이션 | 토스 디자인에 그라데이션 버튼 없음 |
| 보라색 | 프로젝트 색상 시스템에 purple 없음 |
| 직접 스타일링 | Button 컴포넌트 미사용 |

**권장**: 프로젝트 Button 컴포넌트 사용

### 🟡 문제 5: 구분선 문구 ("또는 단일 파일 업로드")

**현재**:
```typescript
<span className="bg-grey-50 px-4 text-sm text-grey-500">
  또는 단일 파일 업로드
</span>
```

**문제점**:
1. "단일 파일 업로드"가 무엇인지 불명확
2. 듀얼 매칭과의 차이점 설명 없음
3. 어떤 것을 선택해야 할지 가이드 없음

**권장 문구**:
```
✓ 문제-해설 동시 작업: 위 영역 사용
✓ 문제만 먼저 등록: 아래 영역 사용
```
