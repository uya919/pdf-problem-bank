# UI/UX 심층 연구 리포트: RegistrationPage vs WorkSessionDashboard

> **작성일**: 2025-12-03
> **목적**: `/` (RegistrationPage)와 `/work` (WorkSessionDashboard) 비교 분석 및 개선 방향 도출

---

## 1. Executive Summary

### 핵심 문제
| 구분 | `/` (RegistrationPage) | `/work` (WorkSessionDashboard) |
|------|------------------------|-------------------------------|
| **주요 행동** | 불명확 (업로드? 세션 시작?) | 명확 (새 세션 시작 / 재개) |
| **정보 밀도** | 높음 (3개 영역 혼재) | 적정 (2개 섹션) |
| **시각적 계층** | 약함 | 강함 |
| **CTA 명확성** | 분산됨 | 집중됨 |

### 결론
`/work` 페이지가 **단일 목표 중심**으로 설계되어 직관적인 반면, `/` 페이지는 **여러 기능이 혼재**되어 사용자가 "다음에 뭘 해야 하지?"라는 의문을 갖게 됨.

## 2. 상세 비교 분석

### 2.1 정보 아키텍처 (IA)

#### `/` RegistrationPage 구조
```
┌─────────────────────────────────────────┐
│ Header: "등록 & 라벨링"                  │
├─────────────────────────────────────────┤
│ [DualDocumentSelector] ← 세션 생성       │
│   - 세션 이름 입력                       │
│   - 문제 문서 선택 리스트                │
│   - 해설 문서 선택 리스트                │
│   - [작업 시작하기] 버튼                 │
├─────────────────────────────────────────┤
│ "또는 파일을 하나씩 등록하세요" 구분선    │ ← ❌ 인지 부하
├─────────────────────────────────────────┤
│ [Drop Zone] ← 파일 업로드                │
├─────────────────────────────────────────┤
│ [처리 중] 문서 리스트                    │
├─────────────────────────────────────────┤
│ [진행 중인 라벨링] 문서 리스트           │
├─────────────────────────────────────────┤
│ [완료된 문서] 문서 리스트                │
└─────────────────────────────────────────┘
```

**문제점:**
1. **6개 영역**이 한 페이지에 혼재
2. DualDocumentSelector와 DropZone이 **경쟁 관계** (둘 다 "시작" 역할)
3. "또는" 구분선이 사용자를 **혼란**스럽게 함
4. 문서 리스트가 3개나 있어 **시선 분산**

#### `/work` WorkSessionDashboard 구조
```
┌─────────────────────────────────────────┐
│ Header: "작업 세션"                      │
│ [새 세션 시작] 버튼  ← 단일 CTA          │
├─────────────────────────────────────────┤
│ [진행 중] 세션 카드들                    │
│   - 상태 아이콘 + 이름 + 진행률          │
│   - [재개] 버튼                          │
├─────────────────────────────────────────┤
│ [완료됨] 세션 카드들 (축소 표시)         │
└─────────────────────────────────────────┘
```

**강점:**
1. **2개 영역**만 존재 (진행 중 / 완료)
2. 단일 주요 CTA: **[새 세션 시작]**
3. 각 세션 카드에 **[재개]** 버튼으로 명확한 다음 행동
4. **진행률 바**로 시각적 피드백

### 2.2 사용자 여정 (User Journey)

#### `/` RegistrationPage - 현재
```
사용자 → 페이지 진입 → ??? (뭘 해야 하지?)
         │
         ├── DualDocumentSelector 보임 (세션 시작?)
         │     └── 하지만 문서 리스트가 비어있을 수 있음
         │
         ├── DropZone 보임 (파일 업로드?)
         │     └── 업로드 후 뭘 해야 하지?
         │
         └── 문서 리스트들 (어떤 게 뭐지?)
               └── 시작하기/계속하기 버튼 클릭
```

#### `/work` WorkSessionDashboard - 현재
```
사용자 → 페이지 진입 → [새 세션 시작] 버튼 명확히 보임
         │
         ├── 세션 없으면: Empty State + [새 세션 시작]
         │
         └── 세션 있으면: 진행 중 카드에서 [재개] 클릭
```

**차이점**: `/work`는 **단일 행동 경로**가 명확

---

### 2.3 시각적 계층 (Visual Hierarchy)

#### `/` 페이지 문제점

1. **타이틀 약함**
   - "등록 & 라벨링" - 두 가지 의미가 합쳐져 불명확
   - 부제목이 기능 나열에 그침

2. **CTA 버튼 분산**
   - DualDocumentSelector의 [작업 시작하기]
   - DocumentCard의 [시작하기] / [계속하기]
   - 어떤 버튼을 눌러야 할지 모호

3. **색상 대비 부족**
   - 모든 카드가 비슷한 스타일
   - 주요 행동이 눈에 띄지 않음

#### `/work` 페이지 강점

1. **헤더에 명확한 CTA**
   - 오른쪽 상단 [새 세션 시작] 버튼 (Solid Blue)

2. **상태별 색상 구분**
   - 라벨링: 파란색
   - 설정: 주황색
   - 매칭: 보라색
   - 완료: 초록색

3. **카드 내 행동 버튼 집중**
   - 진행 중 세션에만 [재개] 버튼

---

### 2.4 인지 부하 (Cognitive Load)

| 요소 | `/` | `/work` |
|------|-----|---------|
| 선택지 수 | 6개 이상 | 2개 |
| 스크롤 필요 | 많음 | 적음 |
| 의사결정 포인트 | 분산 | 집중 |
| 학습 곡선 | 높음 | 낮음 |

## 3. 토스(Toss) UX 원칙 기반 진단

### 3.1 "기본값의 자동화" 원칙 위반

**현재 `/`**:
- 사용자가 문서를 하나씩 선택해야 함
- 어떤 문서가 "문제"이고 "해설"인지 추측해야 함

**개선 방향**:
- 파일명에서 자동 추론 (예: `*_문제.pdf` → 문제, `*_해설.pdf` → 해설)
- 최근 사용 패턴 기반 추천

### 3.2 "한 화면 하나의 목표" 원칙 위반

**현재 `/`**:
- 업로드 + 세션 생성 + 문서 관리가 한 페이지에 혼재

**개선 방향**:
- 각 기능을 분리하거나, 명확한 단계(Step)로 표현

### 3.3 "Progressive Disclosure" 부족

**현재 `/`**:
- 모든 정보가 한 번에 노출

**개선 방향**:
- 필요한 정보만 순차적으로 공개
- 문서 없으면 → 업로드 유도
- 문서 있으면 → 세션 시작 유도

## 4. 개선 제안

### Option A: `/` 페이지 리디자인 (권장)

```
┌─────────────────────────────────────────┐
│ Header: "작업 시작"                      │
│ 부제목: "문제집을 라벨링하고 해설을 연결하세요" │
├─────────────────────────────────────────┤
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  🎯 새 작업 시작하기              │    │
│  │                                 │    │
│  │  [문제 문서 선택 ▼]              │    │
│  │  [해설 문서 선택 ▼]              │    │
│  │                                 │    │
│  │  [▶ 작업 시작]                   │    │
│  │                                 │    │
│  │  ────────────────────────────── │    │
│  │                                 │    │
│  │  📁 새 파일 추가하기              │    │
│  │  [파일 드롭 영역 - 축소형]        │    │
│  └─────────────────────────────────┘    │
│                                         │
├─────────────────────────────────────────┤
│ 진행 중인 세션 (있는 경우만)             │
│  [세션 카드] [재개 →]                   │
└─────────────────────────────────────────┘
```

**핵심 변경:**
1. DualDocumentSelector를 **메인 영역**으로 승격
2. DropZone을 **하위 섹션**으로 축소
3. 문서 리스트 제거 → 드롭다운으로 대체
4. 진행 중 세션만 표시 (완료/처리 중은 별도 페이지)

### Option B: `/work` 페이지로 통합

`/` 페이지를 `/work`로 리다이렉트하고, `/work`에 업로드 기능 추가:

```
┌─────────────────────────────────────────┐
│ Header: "작업 세션"                      │
│ [+ 새 세션] [📁 파일 추가]               │
├─────────────────────────────────────────┤
│ ... (기존 WorkSessionDashboard) ...     │
└─────────────────────────────────────────┘
```

### Option C: 단계별 온보딩 플로우

```
Step 1: 파일 업로드 (최초 1회)
        ↓
Step 2: 문제/해설 페어링
        ↓
Step 3: 작업 시작
```

## 5. 우선순위 권장

| 순위 | 개선 항목 | 영향도 | 난이도 |
|------|----------|--------|--------|
| 1 | `/` 페이지의 DropZone 축소/하단 이동 | 높음 | 낮음 |
| 2 | 문서 리스트 제거 → 드롭다운 전환 | 높음 | 중간 |
| 3 | 헤더에 주요 CTA 집중 | 중간 | 낮음 |
| 4 | 상태별 색상 시스템 적용 | 중간 | 낮음 |
| 5 | 스마트 문서 추천 (파일명 기반) | 낮음 | 높음 |

## 6. 권장 액션 플랜

### Phase 34: 메인 페이지 UX 개선 (예상 3시간)

**34-A: 레이아웃 재구성**
- DualDocumentSelector를 상단 중앙 배치
- DropZone을 하단 축소형으로 변경 (펼침/접힘)

- 상태별 색상 시스템 적용
- Empty State 개선

**34-C: 플로우 단순화**
- 문서 선택 시 드롭다운 사용
- 최근 사용 문서 표시
- 진행 중 세션 우선 노출
